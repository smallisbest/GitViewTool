Command reference

Every subcommand, grouped by purpose. gt --help and gt <command> --help print the same information from the terminal.

Global options

-C, --repo <path>
Run against the repository at <path> instead of the one inferred from the current working directory.
--help
Print top-level help.
--help-all
Print top-level help together with every subcommand's help, recursively.
--version
Print the version string (short).
--version-long
Print version, build number, commit hash, and dependency versions.

Repository setup

gt mkvob [path] [--default-branch <branch>]
Register a Git repository so other gt commands can operate on it. Idempotent — calling it on an already-registered repo just returns the existing registration. If path is omitted, the current working directory is used.
Creates ~/.gt/repos/<id>/ with the repo's metadata database and an initial config file. --default-branch (default: main) sets the fallback branch used by new views that don't specify one.

View management

gt mkview <name> [-b, --branch <branch>] [-m, --mount <path>]
Create a new view. --branch pins the view to that branch (defaults to the per-repo default_cs_branch). --mount overrides the default mount location of ~/.gt/repos/<id>/mounts/<name>. The view is registered but not yet served — run gt mount afterwards.
gt lsview [-l, --long]
List all views in the current repo. -l adds branch, mount path, and creation timestamp.
gt rmview <name> [-f, --force]
Remove a view. Refuses to run if the view is still mounted; -f forces removal and also deletes the overlay directory.
gt setview <name>
Spawn an interactive subshell with the working directory set to the view's mount point and GT_VIEW exported so subsequent gt commands automatically target this view.
gt catcs <name>
Print the view's config spec. In Phase 1 this is a single branch rule.
gt setcs <name> -b, --branch <branch>
Update a view's config spec to pin it to a different branch. Remount the view (gt umount + gt mount) to apply.

Mount lifecycle

gt mount <name> [-d, --debug]
Start the FUSE backend serving a view. Runs in the foreground and blocks until an unmount. -d prints FUSE-level debug output to stderr.
If FUSE-T is missing, gt prints a clear install instruction and exits with code 2.
gt umount <name> [-f, --force]
Unmount a view. Falls back to the /sbin/umount command if the unmount(2) syscall fails. -f passes MNT_FORCE.

Element (file) operations

gt checkout <file> [options] — alias: gt co
Claim a file for edit and materialize its current branch contents into the view's overlay.

-reserved (default) — only one view may hold a reserved checkout of a file at a time.
-unreserved — several views may hold concurrent checkouts; first check-in wins.
-c <comment> — store a comment with the checkout record.
-nc — no comment.
--view <name> — select the view (falls back to GT_VIEW).
gt checkin <file> [options] — alias: gt ci
Convert the view's overlay contents into a real Git commit on the pinned branch, release the checkout, and remove the overlay file. Emits a warning if the branch has moved since the checkout was taken.

-c <comment>, -nc, --view as for checkout.
gt uncheckout <file> [options] — alias: gt unco
Cancel a checkout without creating a commit. The overlay file is removed.

-keep — rename the overlay to <file>.keep before releasing the checkout, preserving a backup.
-rm — discard silently (default).
--view <name> as above.
gt ls [--co, --checkedout] [--view <name>]
List elements in a view. Without flags, lists all files with a * next to any that are checked out. --co shows only checked-out files, prefixed with R (reserved) or U (unreserved).
gt lshistory <file> [-n, --limit <N>] [--view <name>] — alias: gt lsh
Show Git history for a single file on the view's pinned branch. Default limit is 20 commits.
gt diff <file> [-pred, --predecessor] [--view <name>]
Unified diff between the view's overlay and the blob at the branch tip. -pred diffs against the preceding commit's version instead of the current tip.

Information

gt --version
Print the short version string, for example gt 0.1.0+20260422.0115.abc1234.
gt --version-long
Print a multi-line block with version, build number, build timestamp, commit hash and branch, and the embedded versions of libgit2, SQLite, and spdlog.
gt --help / gt --help-all
Top-level help, or a recursive expansion of every subcommand's help text.

Environment variables

VariablePurpose
GT_HOME Root directory for all gt metadata, overlays, and mount points. Defaults to ~/.gt.
GT_REPO Absolute path to a repo that overrides the current working directory (lower priority than -C).
GT_VIEW Default view name used whenever a command accepts --view. Automatically set inside gt setview's subshell.
GT_TRACE_FS When set to a non-empty value, gt mount prints one line per FUSE getattr call to stderr for debugging.

Exit codes

CodeMeaning
0Success
1Command-specific failure (conflict, I/O error, Git error, etc.)
2Environment or argument error (e.g. FUSE-T not installed, no repo found, invalid flag combination)
