Guide — install & first use

From download to your first check-in. Allow about 10 minutes.

Requirements

  • macOS — Apple Silicon or Intel (macOS 13 Ventura or newer)
  • A local Git repository that you already work with
  • About 50 MB of free disk space under your home directory
  • Optional (for gt mount): FUSE-T, a kernel-extension-free user-space filesystem layer (see below)

Step 1 — Install gt

1.1 Download the release for your architecture

Pick the tarball that matches your Mac's CPU from the official download page (the arm64 build is for Apple Silicon, the x86_64 build is for Intel):

cd ~/Downloads
# Apple Silicon:
curl -LO https://smallisbest.github.io/GitViewTool/download/gt-latest-arm64-macos.tar.gz
# Intel:
curl -LO https://smallisbest.github.io/GitViewTool/download/gt-latest-x86_64-macos.tar.gz

1.2 Extract and place the binary

tar -xzf gt-latest-*.tar.gz
sudo mv gt-latest-*/bin/gt /usr/local/bin/
sudo chmod +x /usr/local/bin/gt

If you prefer a per-user install without sudo:

mkdir -p ~/bin
mv gt-latest-*/bin/gt ~/bin/
# then add ~/bin to your PATH if it isn't already
export PATH="$HOME/bin:$PATH"
Check which gt prints the path you installed to.

1.3 First-run Gatekeeper prompt (if any)

The first time you run gt, macOS may show an "unidentified developer" warning. Open System Settings → Privacy & Security and click "Open Anyway". Subsequent launches are unrestricted.

Step 2 — Install FUSE-T (only if you plan to use gt mount)

FUSE-T is a third-party, kernel-extension-free filesystem framework that gt mount uses to serve view contents. Install it once; after that gt mount just works.

2.1 Install via Homebrew

brew install --cask macos-fuse-t/cask/fuse-t

This prompts for your password once to place files under /usr/local. There is no kernel extension, no System-Settings approval step, and no reboot.

Check ls /usr/local/lib/libfuse-t.dylib shows the file.
You only need FUSE-T for gt mount. Every other gt subcommand (including mkvob, mkview, co, ci, lshistory) works without it.

Step 3 — Verify the install

gt --version-long

You should see a block like this:

gt (GitViewTool) 0.1.0+...
  version:      0.1.0
  build:        ...
  built:        ...
  libgit2:      1.9.2
  sqlite:       3.51.0

Step 4 — Create your first view

4.1 Register a Git repository

cd ~/src/myproject
gt mkvob

gt writes a small metadata store under ~/.gt/repos/<id>/. Your repo itself is not modified.

4.2 Create a view pinned to a branch

gt mkview dev --branch main

The view will appear in ~/.gt/repos/<id>/mounts/dev once mounted.

4.3 Mount the view

gt mount dev        # this terminal will block until you unmount

If FUSE-T is not installed you will see a clear instruction. Otherwise the mount succeeds silently.

Check From another terminal, mount | grep gt-dev shows a fuse-t:/gt-dev on … entry.

Step 5 — Your first check-in

5.1 Check out a file

# from another terminal
gt co README.md --view dev -c "my first edit"

The file in the mount is now writable. Other views can't claim a reserved checkout of the same file until you release it.

5.2 Edit through the mount

MOUNT=$(gt lsview -l | awk '/^dev\t/{split($0,a,"mount=");split(a[2],b,"\t");print b[1]}')
$EDITOR "$MOUNT/README.md"

Your edits land in the view's overlay. Other views still see the original branch contents for the same path.

5.3 Check in

gt ci README.md --view dev -c "first edit through gt"

gt commits the overlay contents to the pinned branch, clears the checkout record, and removes the overlay file.

Check Plain git sees the commit: git -C ~/src/myproject log -1 main.

Cleaning up

# in the mount terminal, or from another terminal:
gt umount dev
gt rmview dev       # remove the view entirely (keeps git repo untouched)
