명령어 레퍼런스

용도별로 묶은 모든 서브커맨드 설명. 터미널에서 gt --help 또는 gt <명령> --help로도 같은 내용을 볼 수 있습니다.

전역 옵션

-C, --repo <path>
현재 디렉터리 대신 지정한 경로의 저장소를 대상으로 실행.
--help
상위 help 출력.
--help-all
모든 서브커맨드의 help를 재귀적으로 함께 출력.
--version
짧은 버전 문자열 출력.
--version-long
버전, 빌드 번호, 커밋 해시, 의존성 버전을 여러 줄로 출력.

저장소 설정

gt mkvob [path] [--default-branch <branch>]
Git 저장소를 gt에 등록해 다른 명령이 해당 저장소를 인식하도록 만듭니다. 멱등 — 이미 등록된 저장소에 다시 실행하면 기존 등록 정보만 반환. path 생략 시 현재 디렉터리 사용.
~/.gt/repos/<id>/에 메타데이터 DB와 초기 설정 파일을 생성합니다. --default-branch 기본값은 main이며 view 생성 시 --branch가 지정되지 않으면 이 값이 사용됩니다.

View 관리

gt mkview <name> [-b, --branch <branch>] [-m, --mount <path>]
새 view 생성. --branch는 view가 고정될 브랜치 (생략 시 저장소 설정의 default_cs_branch). --mount는 기본 마운트 위치 (~/.gt/repos/<id>/mounts/<name>)를 override. 생성만 하고 서빙은 gt mount로 별도 시작합니다.
gt lsview [-l, --long]
현재 저장소의 view 목록. -l은 브랜치, 마운트 경로, 생성 시각까지 표시.
gt rmview <name> [-f, --force]
View 제거. 마운트된 상태면 거부하며, -f를 주면 강제로 제거하고 overlay 디렉터리까지 삭제합니다.
gt setview <name>
View의 마운트 디렉터리로 CWD를 옮기고 GT_VIEW를 export한 대화형 서브셸을 실행해, 이후 gt 명령이 해당 view를 자동 대상으로 삼도록 합니다.
gt catcs <name>
View의 config spec 출력. Phase 1에서는 단일 브랜치 규칙.
gt setcs <name> -b, --branch <branch>
View의 고정 브랜치 변경. 반영하려면 gt umount 후 gt mount로 재마운트.

마운트 제어

gt mount <name> [-d, --debug]
View를 FUSE로 서빙 시작. Foreground로 실행되며 umount까지 블록됩니다. -d는 FUSE 레벨 디버그 출력을 stderr로 보냅니다.
FUSE-T 미설치 상태면 명확한 안내를 출력하고 코드 2로 종료.
gt umount <name> [-f, --force]
View 언마운트. unmount(2) syscall이 실패하면 /sbin/umount로 폴백. -f는 MNT_FORCE를 전달.

파일 조작

gt checkout <file> [options] — 별칭: gt co
파일 편집 권한 획득 후 해당 view의 overlay로 현재 브랜치 내용을 구체화.

-reserved (기본) — 해당 파일에 대해 view 하나만 reserved 체크아웃 보유 가능.
-unreserved — 여러 view가 동시에 체크아웃 가능, 먼저 체크인한 쪽이 이김.
-c <comment> — 체크아웃 기록에 코멘트 저장.
-nc — 코멘트 없음.
--view <name> — 대상 view 지정 (GT_VIEW로 대체 가능).
gt checkin <file> [options] — 별칭: gt ci
View의 overlay 내용을 고정된 브랜치에 진짜 Git 커밋으로 만들고, 체크아웃 기록과 overlay 파일을 제거. 체크아웃 이후 브랜치가 움직였으면 경고.

-c <comment>, -nc, --view는 checkout과 동일.
gt uncheckout <file> [options] — 별칭: gt unco
커밋 없이 체크아웃 취소. overlay 파일 제거.

-keep — 취소 전 overlay를 <file>.keep으로 이름을 바꿔 백업.
-rm — 조용히 삭제 (기본).
--view <name> 동일.
gt ls [--co, --checkedout] [--view <name>]
View 내 요소 목록. 플래그 없이 전체 파일을 나열하며 체크아웃된 파일에는 * 표시. --co는 체크아웃된 파일만 표시하며 R(reserved) 또는 U(unreserved) 접두어를 붙입니다.
gt lshistory <file> [-n, --limit <N>] [--view <name>] — 별칭: gt lsh
View가 고정된 브랜치상의 해당 파일 Git 이력. 기본 20개.
gt diff <file> [-pred, --predecessor] [--view <name>]
View의 overlay와 브랜치 tip blob 사이의 unified diff. -pred는 현재 tip 대신 직전 커밋의 버전과 비교.

정보 조회

gt --version
짧은 버전 문자열. 예: gt 0.1.0+20260422.0115.abc1234.
gt --version-long
버전, 빌드 번호, 빌드 시각, 커밋 해시와 브랜치, 내장된 libgit2 / SQLite / spdlog 버전을 여러 줄로 출력.
gt --help / gt --help-all
상위 help 또는 모든 서브커맨드 help의 재귀 확장.

환경변수

변수용도
GT_HOME gt의 모든 메타데이터, overlay, 마운트 루트. 기본값 ~/.gt.
GT_REPO 현재 디렉터리 대신 사용할 저장소 절대 경로 (-C보다 우선순위 낮음).
GT_VIEW --view를 받는 모든 명령의 기본 view. gt setview의 서브셸에서 자동 설정됨.
GT_TRACE_FS 비어있지 않은 값으로 설정하면 gt mount가 FUSE getattr 호출마다 한 줄씩 stderr에 기록 (디버그 용).
