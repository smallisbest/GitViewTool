가이드 — 설치와 첫 사용

다운로드부터 첫 체크인까지. 약 10분 소요.

이 페이지 안에서

요구사항

  • macOS — Apple Silicon 또는 Intel (macOS 13 Ventura 이상)
  • 로컬에 이미 쓰고 있는 Git 저장소
  • 홈 디렉터리에 50 MB 정도의 여유 공간
  • 선택 (gt mount 사용 시): FUSE-T — 커널 확장 없는 사용자 공간 파일시스템 레이어 (아래 설명)

1단계 — gt 설치

1.1 아키텍처에 맞는 릴리스 다운로드

공식 다운로드 페이지에서 본인 Mac의 CPU에 맞는 tarball을 선택하세요 (arm64는 Apple Silicon, x86_64는 Intel):

cd ~/Downloads
# Apple Silicon:
curl -LO https://smallisbest.github.io/gt/download/gt-latest-arm64-macos.tar.gz
# Intel:
curl -LO https://smallisbest.github.io/gt/download/gt-latest-x86_64-macos.tar.gz

1.2 압축 풀고 바이너리 배치

tar -xzf gt-latest-*.tar.gz
sudo mv gt-latest-*/bin/gt /usr/local/bin/
sudo chmod +x /usr/local/bin/gt

sudo 없이 사용자 영역에 설치하려면:

mkdir -p ~/bin
mv gt-latest-*/bin/gt ~/bin/
# ~/bin이 PATH에 없으면 추가
export PATH="$HOME/bin:$PATH"
확인 which gt가 설치 경로를 출력해야 합니다.

1.3 최초 실행 시 Gatekeeper 경고 대응

처음 gt를 실행할 때 macOS가 "알 수 없는 개발자" 경고를 띄울 수 있습니다. 시스템 설정 → 개인정보 보호 및 보안 에서 "그래도 열기"를 한 번 클릭하면 이후에는 뜨지 않습니다.

2단계 — FUSE-T 설치 (gt mount를 쓸 경우만)

FUSE-T는 gt mount가 view 내용을 서빙하는 데 사용하는 커널 확장 없는 사용자 공간 파일시스템 프레임워크입니다. 한 번 설치하면 이후 gt mount는 그대로 동작합니다.

2.1 Homebrew로 설치

brew install --cask macos-fuse-t/cask/fuse-t

파일을 /usr/local 아래에 배치하기 위해 관리자 비밀번호를 한 번 묻습니다. 커널 확장 없음, 시스템 설정 승인 단계 없음, 재부팅 없음.

확인 ls /usr/local/lib/libfuse-t.dylib가 파일을 보여야 합니다.
FUSE-T는 gt mount에만 필요합니다. 그 외 모든 gt 명령 (mkvob, mkview, co, ci, lshistory 등)은 FUSE-T 없이도 동작합니다.

3단계 — 설치 확인

gt --version-long

아래와 같은 블록이 출력되면 성공:

gt (GitViewTool) 0.1.0+...
  version:      0.1.0
  build:        ...
  built:        ...
  libgit2:      1.9.2
  sqlite:       3.51.0

4단계 — 첫 view 만들기

4.1 Git 저장소 등록

cd ~/src/myproject
gt mkvob

gt는 ~/.gt/repos/<id>/ 밑에 작은 메타데이터 저장소를 만듭니다. 원 저장소는 전혀 수정되지 않습니다.

4.2 브랜치에 고정된 view 생성

gt mkview dev --branch main

마운트 후 view는 ~/.gt/repos/<id>/mounts/dev에 노출됩니다.

4.3 View 마운트

gt mount dev        # 이 터미널은 umount 할 때까지 블록됩니다

FUSE-T가 설치되지 않았으면 명확한 안내가 출력됩니다. 설치되어 있으면 조용히 마운트가 완료됩니다.

확인 다른 터미널에서 mount | grep gt-dev를 실행하면 fuse-t:/gt-dev on … 항목이 보입니다.

5단계 — 첫 체크인

5.1 파일 체크아웃

# 다른 터미널에서
gt co README.md --view dev -c "my first edit"

마운트 안의 해당 파일이 쓰기 가능해지고, 다른 view는 release 전까지 reserved 체크아웃을 획득할 수 없습니다.

5.2 마운트 경유 편집

MOUNT=$(gt lsview -l | awk '/^dev\t/{split($0,a,"mount=");split(a[2],b,"\t");print b[1]}')
$EDITOR "$MOUNT/README.md"

편집 내용은 view의 overlay에 저장됩니다. 다른 view에서는 같은 경로가 원래 브랜치 내용으로 보입니다.

5.3 체크인

gt ci README.md --view dev -c "first edit through gt"

gt가 overlay 내용을 브랜치에 커밋하고, 체크아웃 기록을 지우고, overlay 파일을 제거합니다.

확인 git -C ~/src/myproject log -1 main으로 일반 git에서 커밋이 보입니다.

정리

# 마운트 터미널 또는 다른 터미널에서:
gt umount dev
gt rmview dev       # view 완전 제거 (git 저장소는 영향 없음)

  • 사용법 — 병렬 작업, reserved 락, 브랜치 전환, 리모트 pull 등 시나리오 중심의 실전 예시.
  • 명령어 레퍼런스 — 모든 서브커맨드의 플래그와 기본값.
  • 구조 — view, overlay, mount가 어떻게 맞물리는지 내부 구조 설명.
