ClearCase 감각, 내부는 Git.

gt는 일반 Git 저장소를 cleartool 사용자에게 익숙한 개념으로 감쌉니다 — 이름 붙은 view, reserved/unreserved checkout, 그리고 view별 편집 상태를 분리해 주는 동적 마운트. 중앙 VOB 서버도 커널 확장도 필요 없고, 저장소 자체는 전혀 바뀌지 않습니다.

gt가 해결하는 것

독립된 view

각 view는 브랜치에 고정된 영속 작업공간입니다. 같은 브랜치에 올라탄 두 view도 편집 상태, mtime, 체크아웃 목록이 완전히 분리됩니다.

진짜 체크아웃 의미

Reserved 체크아웃은 view 간 배타적 편집 락을 보장하고, unreserved는 동시 작업을 허용하며 먼저 체크인한 쪽이 이깁니다. Git만으로는 할 수 없는 일.

경량 마운트

View의 트리는 사용자 공간 가상 파일시스템으로 제공됩니다. 파일은 지연 로드 — view마다 저장소 전체를 복제하지 않고, 디스크 낭비 0.

view별 mtime 보존

각 view는 파일 수정시간을 독립적으로 관리합니다. view를 전환해도 다른 view의 상태는 건드리지 않습니다.

저장소를 건드리지 않음

gt의 메타데이터는 ~/.gt/에 격리됩니다 — .git/에는 손대지 않음. 기존 git 명령과 IDE 연동 모두 그대로 동작.

진짜 Git 커밋

체크인 시 gt는 view가 가리키는 브랜치에 평범한 Git 커밋을 생성합니다. git log, git push, CI 모두 다른 커밋과 구분하지 않습니다.

한눈에

# 저장소별 1회 등록
gt mkvob

# main 브랜치에 고정된 view 생성 + 마운트
gt mkview dev --branch main
gt mount  dev

# 다른 터미널에서 — 파일 획득, 편집, 체크인
gt co  src/hello.c --view dev -c "adding hello"
$EDITOR ~/.gt/repos/<id>/mounts/dev/src/hello.c
gt ci  src/hello.c --view dev -c "implemented hello"

# 생성된 커밋은 일반 git push로 리모트에
git -C ~/src/myproject push origin main
